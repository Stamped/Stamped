POST comments/create

Description

Add a comment to a specified stamp. The authenticated user must have permission to view and comment on the stamp.

Details

  • OAuth Token Required

Parameters

stamp_id required The id of the stamp being commented on.

blurb required A brief text field containing the comment. This is currently limited to 512 characters. Any embedded mentions will be derived on the server.

Sample Request

POST https://api.stamped.com/v1/comments/create.json
oauth_token=1234567890123456789012&stamp_id=4e259ca532a7ba6cd700000a&blurb=That+looks+awesome.+Thanks+%40robby!

POST comments/remove

Description

Remove a comment. The comment must either be owned by the authenticating user or the authenticating user must own the stamp that was commented on.

Details

  • OAuth Token Required

Parameters

comment_id required The unique id of the stamp.

Sample Request

POST https://api.stamped.com/v1/comments/remove.json
oauth_token=1234567890123456789012&comment_id=4e259ca532a7ba6cd700000b

GET comments/show

Description

Return comments for a given stamp. This includes any restamps that have occurred on the stamp.

Details

  • OAuth Token Required

Parameters

stamp_id required The id of the stamp commented on.

limit optional The maximum number of desired results. Values greater than the default will be ignored.

since optional A UNIX timestamp stating the beginning of the time slice desired. This enables a certain range of comments to be returned.

before optional A UNIX timestamp stating the end of the time slice desired. This enables a certain range of comments to be returned.

Sample Request

GET https://api.stamped.com/v1/comments/show.json?oauth_token=1234567890123456789012&stamp_id=4e259ca532a7ba6cd700000a
